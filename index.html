Javascript Closures

2017-05-03
topic
Javascript, Nodejs, interview

What is a closure?

  • A closure is a function defined within another function that has access to all the variables within the outer function scope even after the outer function returns.
  • It makes it possible for a function to have “private” variables that can maintain their state. (The idea here is similar to instance variables in java)

A classic implementation


function greet(name){
    // private variable
    var message = "Hello " + name + "!!!";

    // a closure : function defined within another function
    function getMessage(){
        // has access to variables withint the outer function scope
        return message;
    }

    // return closure so that it can be accesible outside of this scope;
    return getMessage();
}

console.log(greet('Daredevil')); //Hello Daredevil!!!

Let’s discuss the concept of closure by implementing a counter that can be used from any part of the application.

Approach: Variable outside object

var count = 0;

// functions inside an javascript object
var counter = {
    add: function(){
        count += 1;
    },
    getCount: function(){
        return count;
    },
    reset: function(){
        count = 0;
    }
}

console.log(typeof counter); // object

counter.add();
counter.add();

console.log(counter.getCount()); // 2

count = 0;

console.log(counter.getCount()); // 0

The counter can be reset without calling reset function.

Approach: Variable inside object

// functions inside an javascript object
var counter = {
    count: 0,
    add: function(){
        this.count += 1;
    },
    getCount: function(){
        return this.count;
    },
    reset: function(){
        this.count = 0;
    }
}

console.log(typeof counter); // object

counter.add();
counter.add();

console.log(counter.getCount()); // 2

counter.count = 0;

console.log(counter.getCount()); // 0

The counter can be reset without calling reset function.

Approach: Variable inside a function

// functions inside an javascript object
var counter = new function(){
    // private variable
    var count = 0;

    // a closure : function with another function with access to private variable
    function add(){
        count += 1;
    }

    // another closure
    function getCount(){
        return count;
    }

    // another closure
    function reset(){
        count = 0;
    }

    // all the closures are private by default, make them accessible outside of this scope by returning as a object
    return {
        add: add,
        getCount: getCount,
        reset: reset
    }
};

console.log(typeof counter); // object

counter.add();
counter.add();

console.log(counter.getCount()); // 2

counter.count = 0;

console.log(counter.getCount()); // 2

counter.reset();

console.log(counter.getCount()); // 0

The counter can be reset only by calling reset function. Variable count is truly private and cannot be accessed outside of the function scope.

Bonus: Multiple Counter instances

It is possible to acheive “object oritented” implementation using closures. Javascript design patterns like Module Pattern and Revealing Module Pattern are based on closures.

// functions inside an javascript object
var Counter = function(){
    // private variable
    var count = 0;

    // another way of writing closures that are accessible outside of this scope
    return {
        add: function (){
            count += 1;
        },
        getCount: function(){
            return count;
        },
        reset: function(){
            count = 0;
        }
    }
};

console.log(typeof Counter); // function

var counter1 = new Counter();
var counter2 = new Counter();

console.log(typeof counter1); // object

counter1.add();
counter1.add();
counter2.add();

console.log(counter1.getCount()); // 2
console.log(counter2.getCount()); // 1

counter1.reset();

console.log(counter1.getCount()); // 0
console.log(counter2.getCount()); // 1

Solution to HackerRank Problem - Algorithmic Crush

Solution: Greedy Approach

Test cases #7-13 timed out with this solution.

import java.io.*;
import java.util.*;

public class Solution {

    public static void main(String[] args) {

        Scanner in = new Scanner(System.in);

        int N = in.nextInt();
        int M = in.nextInt();

        long[] arr = new long[N];

        for(int m=0; m < M; m++){
            int a = in.nextInt()-1;
            int b = in.nextInt()-1;
            int k = in.nextInt();

            // update each element between a and b
            while(a<=b){
                arr[a] += k;                 
                a++;
            }
        }

        long max = 0;

        // find the element with maximum value
        for(int i=0; i < N; i++){
            if(arr[i] > max){
                max = arr[i];
            }
        }

        System.out.println(max);
    }
}

Solution: Relative Difference

Instead of storing the actual value of each element in the array, this solution stores the different between current element and previous element.

/* Solution to HackerRank Problem: Algorithmic Crush
 * URL: https://www.hackerrank.com/challenges/crush
 */

import java.io.*;
import java.util.*;

public class Solution {

    public static void main(String[] args) {

        Scanner in = new Scanner(System.in);

        int N = in.nextInt();
        int M = in.nextInt();

        // need an extra elemtent in array to deal with updating element at index b
        long[] arr = new long[N+1];

        for(int m=0; m < M; m++){
            int a = in.nextInt()-1;
            int b = in.nextInt()-1;
            int k = in.nextInt();

            // update element at index a. Difference between further element till index b will be same as k is added to all these elements
            arr[a] += k;

            // update element at index b+1 as difference between element at index b and b+1 has changed after adding k to element at index b
            arr[b+1] -= k;
        }

        long max = 0;
        long sum = 0;

        // find the element with maximum value by summing up the difference
        for(int i=0; i < N; i++){
            sum += arr[i];

            if(sum > max){
                max = sum;
            }
        }

        System.out.println(max);
    }
}

Solution to HackerRank Problem - Nikita and the Game

/* Solution to HackerRank Problem: Nikita and the Game
 * URL: https://www.hackerrank.com/challenges/array-splitting
 */

import java.io.*;
import java.util.*;
import java.text.*;
import java.math.*;
import java.util.regex.*;

public class Solution {

    public static void main(String[] args) {
        Scanner in = new Scanner(System.in);

        int T = in.nextInt();

        for(int t=0; t < T; t++){
            int N = in.nextInt();

            int[] a  = new int[N];

            for(int n=0; n < N; n++){
                a[n] = in.nextInt();
            }

            System.out.println(split(a));
        }
    }

    public static int split(int[] o){
        int length = o.length;

        if(length == 1){
            return 0;
        }

        // array with odd sum cannot be splitted into two parts of equal sum
        int total = sum(o);        
        if(total % 2 == 1){
            return 0;
        }

        int sum = o[0];
        for(int index=1; index < length; index++){

            // iterate till sum of first part is twice the total sum i.e both parts have equal sum
            if(sum * 2 == total){
                int[] o1 = Arrays.copyOfRange(o, 0, index);
                int[] o2 = Arrays.copyOfRange(o, index, length);                

                // consider the part which can be split for maximum times
                return Math.max(split(o1),split(o2))+1;
            }    

            sum += o[index];

        }

        return 0;
    }

    public static int sum(int[] a){
        int sum = 0;

        for(int i : a){
            sum += i;
        }

        return sum;
    }
}

How to make docker work in Windows

2016-08-06
docker, docker toolbox, fix, windows

I tried the new Docket Toolbox for windows. However, the Docker Start Terminal never seemed to work for me. I keep on getting below error, nothing seems to resolve it.

Looks like something went wrong in step 'Checking if machine default exists'...
Press any key to continue...

From the message it looked it something went wrong with the virtual machine, so I decidedto investigate myself.

I executed below command in a cmd to list down the virtual machines available. 

> docker-machine ls

NAME         ACTIVE   DRIVER       STATE     URL   SWARM   DOCKER    ERRORS
my-default   -        virtualbox   Stopped                 Unknown

For some reason, the default VM had not started. I started it manually by executing 

> docker-machine start my-default
Starting "my-default"...
(my-default) Check network to re-create if needed...
(my-default) Waiting for an IP...
Machine "my-default" was started.
Waiting for SSH to be available...
Detecting the provisioner...
Started machines may have new IP addresses. You may need to re-run the `docker-machine env` command.

Waiting for an IP... took a while

Running docker ps or any docker command gave below error 

An error occurred trying to connect: Get http://%2F%2F.%2Fpipe%2Fdocker_engine/v
1.23/containers/json: open //./pipe/docker_engine: The system cannot find the fi
le specified.

To fix this, I had to set appropriate environment variables by executing

> docker-machine env my-default

SET DOCKER_TLS_VERIFY=1
SET DOCKER_HOST=tcp://192.168.99.100:2376
SET DOCKER_CERT_PATH=C:\Users\abhishek_majhi\.docker\machine\machines\my-default

SET DOCKER_MACHINE_NAME=my-default
REM Run this command to configure your shell:
REM     @FOR /f "tokens=*" %i IN ('docker-machine env my-default') DO @%i

Then executed the command suggested in the output.

> @FOR /f "tokens=*" %i IN ('docker-machine env my-default') DO @%i

Now run docker ps or any docker command. It should work now.

> docker ps
CONTAINER ID        IMAGE               COMMAND             CREATED             STATUS              PORTS               NAMES

That’s All Folks!
